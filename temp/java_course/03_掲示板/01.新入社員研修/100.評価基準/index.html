提出物に対するチェックポイント

全体的な提出物に対しては、以下の点についてチェックします。
内容に関してご不明な点がある場合は、お問い合わせください。

全体的なチェックポイント

仕様・機能が全て満たされていること

掲示板・施設予約等のドキュメントにある仕様が全て盛り込まれて、動作することを確認します。
掲示板・施設予約のトップ画面にそれぞれ仕様が記述されていますので、開発前に一通り目を通してください。
また、アプリケーションの画面画像が「画面仕様」となります。
文字色、背景色等の画面レイアウトは変更しないでください。

提出の際の注意が守られていること

それぞれの課題のトップ画面の「研修課題提出の際のご注意」を参照してください。
必要なファイルが添付されていない場合には再提出となります。

添付されたマニュアルが正確であること

マニュアルは第三者が見て、分かりやすいものにしてください。
教育部ではマニュアルに沿ってアプリケーションを実行いたしますので、
ご提出前に一度マニュアルどおり実行して問題なく動作することをご確認ください。

アプリケーションが MVCモデルで作成されていること

掲示板作成時に身に着けていただく MVCモデルでのアプリケーション作成ができていることを確認します。
以下の点になります。

  1. DAO があること
  2. DAO が javax.servlet 等のパッケージを使用していないこと
  3. モデル(ビーン)が存在すること

コーディング方法に問題がないこと

コードについて以下のチェックをいたします。

  1. コーディング規約に則ったコードであること
  2. コードに対して JavaDoc 用コメントが記述されていること
  3. 使用されていない import 文やローカル変数がないこと

セキュリティに関して考察されていること

クロスサイトスクリプティング対策がJSPでされていること。

MVC掲示板チェックポイント

入力エラー時の処理

「*」のついた必須項目が、ひとつでもかけていた場合投稿されず、入力済み文字列が残っていることをチェックします。
この際に、文字色のラジオボタンは選択されていたものが選択されているようにしてください。
パスワードに関しては、入力済み文字列は残らないようにしてください。

削除の処理

違うパスワードを入力したり、何も入力されないまま、削除ボタンを押しても削除されないことをチェックします。

データベース設計に問題がないこと

投稿日時の型が Timestamp 型または Datetime 型であることをチェックします。

クリアボタンが機能している事

クリアボタン押下時に、記入したものが消去できていることをチェックします。

文字数の制限

文字範囲が超えた場合、エラーにならないように処理をしていることをチェックします。

施設予約チェックポイント

データベース設計に問題がないこと

MVC施設予約を作成する際には、データベース設計が終了した時点で設計内容を提出していただき、チェックします。
MVC施設予約DB設計解答.txt

施設数を動的に増やせること

データベースで施設数を増減させた場合に、次回の画面アクセス時から施設数が変更されることをチェックします。

２人のユーザで使用した場合に問題がないこと

施設予約において、２人のユーザが同時に同じ日時・場所の予約をした際に、
先に登録ボタンを押したユーザの予約が上書きされないことを確認します。
