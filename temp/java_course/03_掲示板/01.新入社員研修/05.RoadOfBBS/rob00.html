～掲示板への道～

この研修のゴール

この研修のゴールは、下記の「要求される掲示板の仕様・機能」にて示された機能を満たすMVC掲示板を作成することです。
この資料「～掲示板への道～」に従って、MVC掲示板を作成し、最終的に完成した掲示板を提出してください。

提出物に対するチェックに関しては、チェックポイントをご覧ください。

開発環境基本事項

OS Windows
ブラウザ Chrome 推奨
Servletコンテナ Apache TOMCAT (8.0.26 以降)
データベース MySQL (v 5.6.25 以降)、mysql-connector-java-5.1.35-bin.jar(JDBCドライバ)

  • Tomcat と、MySQL は、「掲示板開始の前に」を参考にしてインストールしておいてください。
  • このマークは、自分で調査してください、という箇所です。調べて理解することが重要です。
    とくに重要なキーワードには赤字にしてあります。
  • 上表で指定しているバージョンが古くなっている場合は、バージョン数値の近いものでかまいません。
  • HTMLの知識がないという方はこちらもご覧下さい。HTMLへの道

要求される掲示板の仕様・機能

  • 入力項目　※印：必須入力項目
    • 名前（テキスト）※
    • メールアドレス（テキスト）
    • 件名（テキスト）
    • メッセージ（テキストエリア）※
    • 文字色（ラジオボタン）
    • パスワード（パスワード）※
    • 投稿（サブミットボタン）
    • リセット（リセットボタン）

  • 入力必須項目は、それらのうち1つでも未入力で投稿ボタンを押下しても、投稿は受け付けない。が、入力済み文字列は残しておく。また、その際にはエラーメッセージを表示する。

  • 文字色選択可能（掲載されたとき、選択された文字色で表現される）。なお、デフォルトの文字色は黒とする。

  • 投稿日時は自動的に付加され、掲載時に表示される。「2003年12月10日(水)午後3時40分50秒」形式の表示とする。

  • 削除機能付き。投稿者自身が投稿時に設定した削除パスワードで削除できる（＝自分の投稿記事しか削除できない）。削除できなかった場合は、エラーメッセージを表示する。

  • 返信機能は不要で、記事編集（訂正）機能も不要。

  • このスクリーンショットのHTMLファイルをダウンロードできます。

研修課題提出の際のご注意

  1. ZIP 形式で圧縮したファイルをメールに添付送信してください。
    圧縮ファイルに含めるのは、ソースコードやクラスファイル一式です。
    基本的なディレクトリ構成としては、「～掲示板への道～」でのサンプルにもあるように、
    %WEBAPPLI%
   |
   +---\WEB-INF
           |
           +---\classes
           |
           +---\lib
     としていただき、そのまま %WEBAPPLI% ごと圧縮ファイルにしてください。

    GMail を使っている場合、lib フォルダに jar ファイルが入っていると、送信エラーで
    メールが送れないことがあります。その場合は、lib ファイルの中を空にして送信してください。

  2. 掲示板のために使うデータベース名を、

    × CREATE DATABASE BBS;
    とするのではなく、
    ◎ CREATE DATABASE WAKAI_BBS;
    というように、「誰のものか」 がわかるようにしておいてください。

    ソースファイルなどにはこのような措置は不要です。
    （たとえば、YamadaBbsServlet.java等にはしないでください。）

  3. マニュアルを付録しておいてください。

    ファイル    「例：MVC掲示板.txt」を参考にしてマニュアルを作成してください。

    • WORDやEXCELファイル形式ではなく、テキストファイルでお願いいたします。
    • テーブル作成時に使用した CREATE文 や、
      テーブルの初期データ用の INSERT文など（あれば）をそのまま書いておいてください。

  4. コーディング規約に則ってコーディングしてください。
    １行の行数や、インデントなどに注意します。 「インデント例.java」をご覧ください。

    見にくいソースコードは、ロジックもまたスマートではありません。
