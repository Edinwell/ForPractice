～施設予約への道～

この研修のゴール

この研修のゴールは、下記の「要求される施設予約の仕様・機能」にて示された機能を満たすMVC施設予約を作成することです。

この資料「～施設予約への道～」は「～掲示板への道～」と違って、細かい説明はありません。
いままでMVC掲示板作成で身に着けた技術を応用すれば充分作れるはずです。
MVC施設予約を作成し、最終的に完成した施設予約を提出してください。

提出物に対するチェックに関しては、チェックポイントをご覧ください。

要求される施設予約の仕様・機能

施設予約システムの仕様を、画面例で説明します。

1.初期画面

初期画面です。
社員番号を入力して、ログインボタンを押下します。 社員番号はデータベースで社員情報テーブルから照会します。

2.ログイン

ログインしました。
施設を予約可能な表示になります。

3.予約手続き

大講堂を9時から12時まで予約しようとチェックをつけ、 予約ボタンを押下します。

4.予約完了

予約を完了しました。

5.予約取消し手続き

実のところ、大講堂は２時間で済むので、11時のは取り消すことにしました。 予約取消ボタンを押下します。

6.完了

意図したとおりに予約が出来ました。ログアウトボタンを押下します。

7.ログアウト

ログアウトしました。
若井社員のPC上では、若井社員の社員番号「0106」が残っています。

8.別の人も利用

印差社員も、自分のPC上で予約をしてみたいと思います。 印差社員の社員番号は「1111」です。

9.ログイン

印差社員もログインしました。
すでに大講堂は他の社員（＝若井社員）によって9時から10時まで予約が取られています。

10.予約手続き

印差社員は会議室を利用することにします。

11.予約完了

印差社員が予約した部分は、印差社員のみ予約取り消しは出来ますが、
他者の予約した部分は取消すことは出来ません。

12.予約不可能

過去の日付(1999年になってます)の予約は出来ません。意味がないからです。
ただしそれでも表示のみは出来ます。過去の予約状況を確認したい場合に対応してます。

ということで、基本的な仕様は以上のようにしてください。
尚、予約対象の施設ですが、データベースを変更するのみで増減可能とします。
予約可能な時間帯は変更可能であることが望ましいですが、９時～２１時の固定でも構いません。

誰が、何年何月何日の何時から何時まで、どの施設を予約できるかが この施設予約システムです。
予約ができる（空室である）か、（埋まっているので）できないか。 そして自分の予約は取り消せるが、他者の予約は取り消せない。
そこがポイントです。

このスクリーンショットのHTMLファイルをダウンロードできます。

この研修の進め方

初めに施設予約で必要となるデータベースを設計して、一度ご提出ください。
データベース設計に問題がなければ、画面仕様に沿って開発を進めていただきます。
データベース設計提出時のフォーマットに関しては、

  1. テーブル名
  2. カラム名
  3. 型（長さ）
  4. 制約
がわかればどのような形式でも構いません。掲示板の提出マニュアルを参考にしてください。

研修課題提出の際のご注意

基本的にMVC掲示板の提出時と同じです。
アプリケーションのフォルダごと圧縮ファイルにまとめていただき、マニュアルを添付して提出してください。
