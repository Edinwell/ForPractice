この課題のゴール

この課題のゴールは仕様どおりのCUIアプリケーションを作成することです。
やさしいJava、独習Java で学んだ基礎知識が身についていて、応用できるかどうかを確認します。

尚、どうしてもアプリケーションが作成できない場合は、研修担当に相談の上、
作成STEP例 をもらって、STEPに沿って実装してください。

CUI課題仕様

■作成課題

ユーザの入力を受け付けて、『○』や『×』を出力する以下のようなアプリケーションを作成してください。

※┌ などの線は「けいせん」を変換すると表示できます。

┌─┬─┬─┐
│○│×│×│
├─┼─┼─┤
│　│○│○│
├─┼─┼─┤
│○│　│×│
└─┴─┴─┘
３×３のマス目があり、そこに『○』または『×』が表示されます。
ユーザが「1,2」というように２つの整数を入力した場合に
上から２番目の左から３番目のマスに『○』を表示する(Javaでは0からカウントするため)、というように判断します

■課題の内容

実行例を見ていただくと、『○』と『×』が交互に出力されていること、
指定した２つの数字が縦軸インデックス、横軸インデックスに使用されていることがわかると思います。

細かい仕様は以下のとおりです。

  1. 「[○の番] 縦と横のインデックスを半角スペース区切りで指定してください。」と表示します。
    （『×』の順番の場合は、［×の番］と表示します。）
  2. 入力を受け付けます。
  3. ブランク区切りで０～２の範囲の２つの整数（縦軸インデックス・横軸インデックス）を入力します。
  4. 指定された位置に『○』を表示します。（『×』の順番の場合は『×』）
  5. ９つのマスが全て埋まるまで、○と×交互に「1 ～ 4」の処理を繰り返します。
  6. ９つのマスが全て埋まったら「ゲームを終了します。」と出力して終了します。
入力の際に、
  • 数字以外が入力された場合、エラーを表示します。
  • 入力された整数の数が２個より小さい場合、エラーを表示します。
  • 範囲外の場所が指定された場合、エラーを表示します。
  • 『○』または『×』が既に表示されている場所が指定された場合、エラーを表示します。
メッセージの仕様は以下とします。

入力を促すメッセージ [○の番] 縦と横のインデックスを半角スペース区切りで指定してください。
終了メッセージ ゲームを終了します。
数字以外が入力された場合 エラー：整数を入力してください。
入力数が２より小さい場合 エラー：整数は２つ入力してください。
範囲外の整数が指定された場合 エラー：整数は0 ～2の間で指定してください。
既に入力がある場所が指定された場合 エラー：指定された場所は既に入力されています。

※３つ揃った時点で勝ちになる等のゲームロジックの実装は必要ありません。
