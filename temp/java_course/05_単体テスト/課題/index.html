単体テスト課題

課題回答手順

この課題では、以下の段階を踏んで「MVC掲示板」の、単体テストの作成を実施していきます。
課題が１つ終了するごとに、担当者にレビューをしてもらい、OKが出たら次に進んでください。

この課題でのチェック項目は以下になります。
レビューポイント

ユーティリティクラスの直交表作成

初めに、MVC掲示板で使用しているユーティリティクラスをテストします。
デフォルトで作成した場合、HTML用のエスケープを行うメソッドがあると思いますので、それを対象にします。
自分で作成したユーティリティメソッドがある場合は、その直交表も作成してください。

直交表の書き方が分からない人は、ここで復習してください。

ユーティリティクラスのテストクラス作成

作成した直交表を元に、JUnitを使用して、単体テストを実装してください。

JUnitの使用方法が分からない人は、ここで復習してください。

DAO クラスの直交表作成

MVC掲示板のDAOクラスの直交表を作成してください。
最初に、使用するデータを考慮に入れると、後々のテストが楽になります。

DAOの直交表の書き方が分からない人は、ここで復習してください。

DAO クラスのテストクラス作成

作成した直交表を元に、JUnit、DBUnitを使用して、単体テストを実装してください。

DBUnitの使い方が分からない人は、ここで復習してください。

サーブレットクラスの直交表作成

MVC掲示板のサーブレットクラスの直交表を作成してください。
サーブレットクラスの中で呼び出すユーティリティメソッドや、
DAOメソッドは、全てモックを使用するという考え方で作成します。

サーブレットの直交表の書き方が分からない人は、ここで復習してください。

サーブレットクラスのテストクラス作成

作成した直交表を元に、JUnit、djUnit、サーブレットモックを使用して、単体テストを実装してください。

サーブレットモックの使い方が分からない人は、ここで復習してください。
djUnitの使い方が分からない人は、ここで復習してください。
